# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

Tauri v2 plugin monorepo (pnpm workspace + Cargo workspace) for privacy-first, local-first plugins. Currently contains the `material-you` plugin (Android Material You dynamic colours). See `PLUGIN_TEMPLATE.md` for the canonical new-plugin structure.

**`AGENTS.md` is the canonical source of repository rules** — spelling, commit conventions, PR format, code organisation, plugin development patterns, and release process. Always consult it for authoritative guidance.

## Essential Commands

```bash
# Setup
pnpm install
pnpm hooks:install          # configure git hooks

# Build all JS plugins
pnpm build

# Run all tests
pnpm test                   # runs both test:rust and test:js
cargo test --workspace      # Rust only
pnpm test:js                # JS only (vitest)

# Run a single JS test file
npx vitest run tests/material-you-guest-js.test.js

# Linting and formatting
cargo fmt --all             # format Rust
cargo fmt --all --check     # check Rust formatting
cargo clippy --workspace --all-targets -- -D warnings
pnpm lint                   # ESLint
pnpm format:check           # Prettier check
pnpm format                 # Prettier write

# Run the example app
pnpm tauri                  # runs the api example via Tauri CLI

# Release management
pnpm covector change        # create a change file
```

## Localisation Requirement

**All code, comments, commits, PR descriptions, and documentation MUST use Canadian English spelling.** Examples: `colour` not `color`, `centre` not `center`, `neighbour` not `neighbor`, `cancelled` not `canceled`.

## Commit and PR Conventions

- **Commits:** Conventional Commits (`feat:`, `fix:`, `docs:`, `chore:`, `test:`). Body should use `**What changed**`, `**Why**`, `**Scope**` structure with **bold labels** (no markdown headings). Release commits start with `Release` (no type prefix). Prefer heredoc (`git commit -F <file>`) for markdown-heavy bodies.
- **PR titles:** Human-readable behaviour summaries. No Conventional Commit prefixes. Start with capital letter.
- **PR descriptions:** Use `## Summary` and `## Test plan` sections. Test plan uses checklist bullets (`- [x]`/`- [ ]`).
- **PR labels:** Required primary category (`bug`, `enhancement`, `documentation`, `testing`, `ci`, `build`, `release`, `chore`). Do not use aliases like `feat` or `fix`.

## Architecture

### Workspace Layout

- `plugins/` — Tauri v2 plugins (each has Rust crate + JS package + optional Android/iOS)
- `shared/` — Shared build tooling (Rollup config used by all plugins)
- `examples/api/` — Example Tauri app for integration testing
- `.changes/` — Covector change files and release config
- `tests/` — Workspace-level JS tests (vitest)

### Plugin Dual-Package Structure

Each plugin is both a Rust crate (`tauri-plugin-{name}`) and an npm package (`@liminal-hq/plugin-{name}`):

- **Rust side:** `src/lib.rs` (init + command handler registration), `commands.rs`, `error.rs`, `models.rs`, `mobile.rs`/`desktop.rs` (platform-specific). `build.rs` registers commands and sets up the IIFE global API script.
- **JS side:** `guest-js/index.ts` → built via Rollup into `dist-js/` (ESM + CJS) and `api-iife.js` (for Tauri's webview). Uses `invoke('plugin:{name}|{command}')` pattern.
- **Permissions:** `permissions/default.toml` defines allowed commands. `autogenerated/` is created by the build.
- **Android:** Kotlin plugin in `android/src/main/java/com/plugin/{name}/`

### Shared Rollup Config

Plugins import `createConfig(pluginName)` from `shared/rollup.config.js`. This generates ESM, CJS, and IIFE outputs with TypeScript compilation and declaration generation.

### Versioning

Covector manages releases. Rust and JS versions are kept in sync per plugin. Both package entries (Rust + JS) must exist in `.changes/config.json`.

## Key Constraints

- **No barrel files** — import directly from concrete module paths, never use `index.ts` re-export barrels.
- **Tauri v2 only** — do not use v1 patterns. Capabilities replace `allowlist`. Plugins need explicit permission grants.
- **Android permissions** — plugins must own permissions via `tauri_plugin::mobile::update_android_manifest()` in `build.rs`. Never require consuming apps to manually edit Android manifests.
- **Do not push** — never push changes (especially force pushes) unless explicitly requested.

## Toolchain Versions

- Rust: 1.93.0 (pinned in `rust-toolchain.toml`, includes `rustfmt` + `clippy`, targets: x86_64-linux-gnu + aarch64-linux-android)
- Node.js: >=24.14.0 <25 (pinned in `.node-version`)
- pnpm: 10.28.2
- Android NDK r28 (for mobile development, available in devcontainer)
